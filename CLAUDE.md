# CLAUDE.md — duplicacy-cli-cron

## Overview
Docker container that runs Duplicacy CLI backups on a cron schedule to S3-compatible storage with AES-256-GCM encryption, automatic pruning, integrity checks, and Telegram notifications via Shoutrrr.

## Tech Stack
- Shell (Bash scripts)
- Duplicacy CLI (backup engine)
- Docker (Alpine-based, multi-arch: amd64, arm64, armv7)
- Shoutrrr (notifications)
- S3-compatible storage (Garage, MinIO, AWS S3, Backblaze B2)

## Development
```bash
# Build the container
docker build -t duplicacy-cli-cron .

# Run with Docker Compose
docker-compose up -d

# Run tests
cd tests && ./run-tests.sh
```

## Architecture
- `entrypoint.sh` — Container entry point, sets up cron jobs
- `scripts/` — Core backup logic
  - `dual-executor.sh` — Shared locking, backup, prune, and notification handler
  - `exhaustive-prune.sh` — Weekly deep prune that reclaims storage by scanning all chunks
  - `monthly-integrity-check.sh` — Monthly verification of all backup chunks
- `config/` — Configuration templates and filter files
- `tests/` — Test suite
- `docs/` — Documentation and guides

## Key Rules
- Docker image published to Docker Hub as `drumsergio/duplicacy-cli-cron` with semver tags
- Never use `:latest` tag; always pin specific versions
- Per-repo lock files with automatic timeout kills after `MAX_RUNTIME_HOURS`
- Stagger cron schedules across servers to avoid storage contention
- All encryption passwords are per-repo; never share or hardcode them
